  - In the existing onKeyDown handler (which already handles ArrowUp/ArrowDown from Plan 01), add a Tab case:
    - `e.preventDefault()` to prevent focus change
    - Call `getCompletions(value)` where value is the current input
    - If exactly 1 match: call `onChange(matches[0])` to auto-complete inline, clear suggestions
    - If 2+ matches: `setSuggestions(matches)` to show chips
    - If 0 matches: clear suggestions
  - Clear suggestions whenever: onChange is called with a new value (user types), onSubmit is called, or input becomes empty. Add a useEffect or inline logic: when value changes, if getCompletions(newValue) produces different results, update suggestions; if value is empty, clear suggestions.
  - Actually simpler approach: clear suggestions on every onChange call and on submit. Only set suggestions on Tab press with multiple matches. This avoids stale chip state.
  - Add `onSelectSuggestion` handler: calls `onChange(command)` and clears suggestions
  - Render the Autocomplete component after the form element (outside the form, inside a wrapping fragment or div). Pass suggestions and onSelectSuggestion.
  - Wrap the entire return in a div (instead of just the form) so the chips appear directly below the input bar. The outer div should have the same border-t and bg-console-surface styling, and the form should lose its border-t (move it to the wrapper).

  In terminal-shell.tsx:
  - No changes needed (input component handles everything internally now).
