<task type="auto">
  <name>Task 1: Extend terminal types and reducer with history state and navigation</name>
  <files>src/lib/terminal/types.ts, src/lib/terminal/reducer.ts</files>
  <action>
  In types.ts:
  - Add to TerminalState: `commandHistory: string[]` (list of submitted commands), `historyIndex: number` (-1 = not browsing, 0+ = position from start), `savedInput: string` (preserves typed text when entering history mode)
  - Add to TerminalAction union: `{ type: "HISTORY_UP" }` and `{ type: "HISTORY_DOWN" }`

  In reducer.ts:
  - Update `initialState` to include `commandHistory: []`, `historyIndex: -1`, `savedInput: ""`
  - Add HISTORY_UP case: If commandHistory is empty, return state. If historyIndex is -1 (first press), save current inputValue to savedInput and set historyIndex to commandHistory.length - 1. Otherwise decrement historyIndex (min 0). Set inputValue to commandHistory[newIndex].
  - Add HISTORY_DOWN case: If historyIndex is -1, return state. Increment historyIndex. If newIndex >= commandHistory.length, reset historyIndex to -1 and restore inputValue from savedInput. Otherwise set inputValue to commandHistory[newIndex].
  - In SUBMIT_COMMAND case: After parsing, push trimmed command to commandHistory (skip if last entry is identical -- consecutive dedup). Reset historyIndex to -1 and savedInput to "". For CLEAR command, also reset historyIndex and savedInput but keep commandHistory intact (clear stores in history per research recommendation).
  - In SET_INPUT case: Reset historyIndex to -1 (user is manually typing, exit history mode). Do NOT reset savedInput here.
  </action>
  <verify>Run `npx tsc --noEmit` -- no type errors. Manually verify reducer logic handles: empty history (no-op on arrow up), single entry navigation, boundary wrapping, consecutive dedup.</verify>
  <done>TerminalState has commandHistory/historyIndex/savedInput fields. Reducer handles HISTORY_UP, HISTORY_DOWN correctly. SUBMIT_COMMAND stores non-consecutive-duplicate commands. SET_INPUT resets history browsing.</done>
</task>
