Structure (follows research code example closely):
```
<div ref={containerRef} onClick={handleContainerClick} className="flex flex-col h-full bg-console-bg">
  {/* Header */}
  <div className="shrink-0 px-4 py-2 border-b border-console-border bg-console-surface flex items-center gap-2">
    <span className="size-2 rounded-full bg-glow-green" />
    <span className="text-glow-green text-sm font-bold tracking-wide">Yogi Dev Console v1.0</span>
  </div>

  {/* Scrollable output */}
  <div className="flex-1 overflow-y-auto p-4 space-y-4">
    {state.history.map(entry => <TerminalOutputEntry key={entry.id} entry={entry} />)}
    <div ref={outputEndRef} />
  </div>

  {/* Input */}
  <TerminalInput ... />
</div>
```

Key behaviors:
- Auto-scroll: `useEffect` that calls `outputEndRef.current?.scrollIntoView({ behavior: "smooth" })` when `state.history.length` changes
- Focus management: `handleContainerClick` finds the input element in the container and calls `.focus()` (Pitfall 1 from research)
- Wire reducer: `dispatch({ type: "SET_INPUT", value })` for onChange, `dispatch({ type: "SUBMIT_COMMAND", command })` for onSubmit
- Resume side effect: `useEffect` that checks the latest history entry -- if its output type is "resume", trigger the download via creating a temporary `<a>` element pointing to `/resume.pdf` with download attribute `Yoganand_Govind_Resume.pdf` (Pattern 4 from research)
