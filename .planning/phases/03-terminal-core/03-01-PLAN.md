  <files>src/lib/terminal/types.ts, src/lib/terminal/commands.ts</files>
  <action>
Create `src/lib/terminal/types.ts` with:

1. `CommandOutput` discriminated union with these variants:
   - `{ type: "welcome" }`
   - `{ type: "help" }`
   - `{ type: "about" }`
   - `{ type: "skills" }`
   - `{ type: "experience" }`
   - `{ type: "projects" }`
   - `{ type: "architecture" }`
   - `{ type: "achievements" }`
   - `{ type: "certifications" }`
   - `{ type: "contact" }`
   - `{ type: "resume" }`
   - `{ type: "error"; input: string }`

2. `TerminalEntry` interface: `{ id: string; command?: string; output: CommandOutput; timestamp: number }`
   - `command` is optional (undefined for welcome message)

3. `TerminalState` interface: `{ history: TerminalEntry[]; inputValue: string }`

4. `TerminalAction` discriminated union:
   - `{ type: "SUBMIT_COMMAND"; command: string }`
   - `{ type: "SET_INPUT"; value: string }`
   - `{ type: "CLEAR" }`

5. Export `CommandName` type derived from command registry keys.

Create `src/lib/terminal/commands.ts` with:

1. `COMMANDS` object using `as const satisfies Record<string, { description: string }>` pattern (matches project convention from portfolio.ts):
   ```
   help: "Show available commands"
   about: "Professional summary"
   skills: "Technical skills by category"
   experience: "Career timeline"
   projects: "Project showcase"
   architecture: "System architecture diagram"
   achievements: "Key achievements with metrics"
   certifications: "Professional certifications"
   contact: "Contact information"
   resume: "Download resume PDF"
   clear: "Clear terminal"
   ```

2. `CommandName` type: `keyof typeof COMMANDS`

3. `parseCommand(input: string): CommandOutput` function:
   - Trim and lowercase input
   - If input matches a key in COMMANDS, return `{ type: commandName }`
   - Special case: "clear" is NOT returned as CommandOutput -- return it anyway; the reducer handles the clear action separately
   - If no match, return `{ type: "error", input: trimmed }`

Export COMMANDS, parseCommand, and CommandName.
  </action>

  <done>Types file exports CommandOutput, TerminalEntry, TerminalState, TerminalAction. Commands file exports COMMANDS registry, parseCommand function, and CommandName type. All compile cleanly.</done>
</task>

<task type="auto">
  <name>Task 2: Terminal state reducer</name>
  <files>src/lib/terminal/reducer.ts</files>
  <action>
Create `src/lib/terminal/reducer.ts` with:

1. `createEntry(output: CommandOutput, command?: string): TerminalEntry` helper:
   - Generates unique id via `Date.now().toString(36) + Math.random().toString(36).slice(2)` (no external deps)
   - Sets timestamp to `Date.now()`

2. `initialState: TerminalState`:
   - `history` contains one entry: a welcome message (`createEntry({ type: "welcome" })`)
   - `inputValue` is `""`

3. `terminalReducer(state: TerminalState, action: TerminalAction): TerminalState`:

   Handle each action:

   **SUBMIT_COMMAND:**
   - Trim the command string; if empty, return state unchanged (prevent empty submissions -- Pitfall 3 from research)
   - Call `parseCommand(trimmed)` to get output
   - If the parsed output type is "clear": return state with `history` containing only a new welcome entry, and `inputValue: ""`
   - Otherwise: append new `TerminalEntry` (with `command: trimmed` and the parsed output) to history, reset `inputValue: ""`

   **SET_INPUT:**
   - Return state with updated `inputValue`

   **CLEAR:**
   - Return state with `history` containing only a new welcome entry, `inputValue: ""`

Export `terminalReducer` and `initialState`.

Important: Do NOT import React -- this is a pure function module. Do NOT handle side effects (resume download, scrolling) in the reducer -- those are handled by UI components.
  </action>
  <verify>Run `npx tsc --noEmit` -- no type errors. Verify the reducer is a pure function with no React imports: `grep -c "import.*react" src/lib/terminal/reducer.ts` should return 0.</verify>
  <done>Reducer handles SUBMIT_COMMAND (with parseCommand integration), SET_INPUT, and CLEAR. Initial state includes a welcome entry. Empty input is guarded. Clear resets to welcome-only state.</done>
</task>
