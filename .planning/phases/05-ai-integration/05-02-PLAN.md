   - Import `portfolio` from `@/data/portfolio`
   - Export `buildSystemPrompt(): string`
   - Serialize ALL portfolio sections into a structured system prompt
   - Follow the persona rules from CONTEXT.md:
     - Name: "Yogi AI -- Portfolio Assistant"
     - Speaks in first person as Yoganand
     - Tone: professional, concise, technical, helpful, slightly conversational
     - 60% developer console feel, 40% conversational AI
   - Response guidelines in the prompt:
     - Keep responses 6-12 lines
     - Short conversational intro + structured content (bullets) + optional call-to-action
     - Bold labels sparingly (e.g., "Project: Autowire.ai")
     - No emojis, no heavy markdown headers
     - For architecture/system design questions: include ASCII flow diagrams
     - For skills/experience/about: use bullets, no diagrams
     - Suggest related topics at end of responses
   - Off-topic handling: "Politely redirect: I'm focused on Yoganand's portfolio -- try asking about his projects or experience!"
   - Special commands section: list the curated responses for "why hire yoganand", "start project", "design system", "fun fact", "career journey", "learning now", "autowire architecture"
   - Serialize portfolio data concisely:
     - About: name, role, summary (trimmed)
     - Skills: category + skill names joined
     - Experience: title, company, period, description, contributions
     - Projects: name, subtitle, description, features, architecture flow, tech stack, URL
     - Achievements: list of descriptions
     - Certifications: name + status
     - Contact: label + URL

3. Create `src/app/api/chat/route.ts`:
   - Import `streamText`, `UIMessage`, `convertToModelMessages` from 'ai'
   - Import `anthropic` from '@ai-sdk/anthropic'
   - Import `buildSystemPrompt` from '@/lib/ai/system-prompt'
   - Import `env` from '@/lib/env' (this triggers env validation on first request)
   - Export `POST` async function:
     - Parse body: `const { messages }: { messages: UIMessage[] } = await req.json()`
     - Safety guard: limit to last 20 messages to prevent token explosion: `messages.slice(-20)`
     - Call `streamText`:
       - `model: anthropic('claude-haiku-3-5-20241022')` -- use dated model ID for determinism
       - `system: buildSystemPrompt()`
       - `messages: await convertToModelMessages(messages.slice(-20))`  (note: convertToModelMessages may or may not be async -- check at implementation time, if not async remove await)
       - `temperature: 0` (AI-04: minimize hallucination)
       - `maxOutputTokens: 1024` (keep responses concise)
     - Return `result.toUIMessageStreamResponse()`
   - Do NOT add rate limiting here yet -- that's Plan 03
   - The `@ai-sdk/anthropic` package auto-reads `ANTHROPIC_API_KEY` from env

4. Create `.env.example`:
   ```
   # Anthropic API Key (required for AI chat)

   ```

5. Update `.gitignore` -- verify `.env.local` and `.env` are already in gitignore. If not, add them. Do NOT add `.env.example` to gitignore (it should be committed).

IMPORTANT NOTES:
