  <name>Task 1: Create rate limiter and integrate into route handler</name>
  <files>
    src/lib/ai/rate-limit.ts
    src/app/api/chat/route.ts
  </files>
  <action>
1. Create `src/lib/ai/rate-limit.ts`:
   - In-memory sliding window rate limiter using a `Map<string, number[]>`
   - Configuration: `WINDOW_MS = 60_000` (1 minute), `MAX_REQUESTS = 10` (10 per minute per IP)
   - Export `rateLimit(identifier: string): { success: boolean }`
   - Logic:
     - Get timestamps array for identifier (default empty)
     - Filter out expired timestamps (older than WINDOW_MS)
     - If valid count >= MAX_REQUESTS, return `{ success: false }`
     - Otherwise push current timestamp, return `{ success: true }`
   - Add periodic cleanup: `setInterval` every 60 seconds to prune empty entries from the Map (prevents memory leak)
   - This is a module-level Map -- persists across requests in the same serverless instance (sufficient for portfolio site)

2. Update `src/app/api/chat/route.ts`:
   - Import `rateLimit` from `@/lib/ai/rate-limit`
   - Add rate limit check at the TOP of the POST handler, before parsing body:
     ```
     const ip = req.headers.get('x-forwarded-for') ?? req.headers.get('x-real-ip') ?? 'unknown';
     const { success } = rateLimit(ip);
     if (!success) {
       return Response.json(
         { error: 'Too many requests. Please try again in a moment.' },
         { status: 429 }
       );
     }
     ```
   - Wrap the streamText call in a try/catch:
     - On error: return `Response.json({ error: 'Something went wrong. Please try again.' }, { status: 500 })`
     - Log the error server-side: `console.error('Chat API error:', error)`
   - Do NOT expose error details to client (no stack traces, no model names in error responses)
  </action>
  <verify>
    - `npx tsc --noEmit` passes
    - File exists: `src/lib/ai/rate-limit.ts`
    - Route handler has rate limit check and try/catch error handling
  </verify>
  <done>Rate limiter created with 10 req/min sliding window. Route handler checks rate limit before processing and catches errors gracefully.</done>
