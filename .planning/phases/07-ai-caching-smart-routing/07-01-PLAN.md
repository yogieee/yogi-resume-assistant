Create a keyword-based intent classifier. No ML, no embeddings -- pure deterministic pattern matching.

Define the Intent type:
```typescript
export type Intent =
  | "about"
  | "skills"
  | "experience"
  | "projects"
  | "architecture"
  | "achievements"
  | "certifications"
  | "contact"
  | "hire"
  | "novel";
```

Define IntentRule interface with `intent`, `keywords` (string[]), and optional `patterns` (RegExp[]).

Create INTENT_RULES array covering all topics. Use the research examples as baseline but ensure thorough keyword coverage:
- about: "about", "who", "background", "introduce", "yourself", "summary", "bio", "tell me about"
- skills: "skills", "technologies", "tech stack", "languages", "tools", "frameworks", "know", "proficient"
- experience: "experience", "work", "career", "job", "employment", "company", "mphasis", "curam", "ibm"
- projects: "projects", "autowire", "portfolio", "built", "created", "saas", "platform"
- architecture: "architecture", "design system", "serverless", "cloud", "infrastructure", "cdk", "aws design"
- achievements: "achievements", "accomplishments", "highlights", "proud"
- certifications: "certifications", "certified", "aws cert", "credentials"
- contact: "contact", "email", "linkedin", "github", "reach", "connect"
- hire: "hire", "collaborate", "work together", "freelance", "consulting", "start project", "why hire"

Add regex patterns for natural language variants:
- skills: /what (do|does|can) (you|yogi) (know|use|work with)/i
- projects: /what (have|has) (you|yogi) (built|made|created)/i
- hire: /why (should|would).*(hire|choose)/i
- experience: /where (have|has|do|does) (you|yogi) work/i

Export `classifyIntent(message: string): Intent` function that:
1. Normalizes input (lowercase, trim)
2. Iterates INTENT_RULES checking keywords first (includes check), then patterns
3. Returns matched intent or "novel"

IMPORTANT: Keywords are checked with `normalized.includes(kw)` so multi-word keywords like "tech stack" work. Order rules from most specific to least specific to avoid false matches.

**Also export `textToStreamChunks` utility** -- this converts static response strings into the LanguageModelV3StreamPart[] format that `simulateReadableStream` expects in the caching middleware. Plan 03 needs this to pre-seed the cache with static responses.

```typescript
import type { LanguageModelV3StreamPart } from "@ai-sdk/provider";

export function textToStreamChunks(text: string): LanguageModelV3StreamPart[] {
  const words = text.split(/(\s+)/);
  const chunks: LanguageModelV3StreamPart[] = words.map((word) => ({
    type: "text-delta" as const,
    textDelta: word,
  }));
  chunks.push({
    type: "finish" as const,
    finishReason: "stop" as const,
    usage: { promptTokens: 0, completionTokens: 0 },
  });
  return chunks;
}
```
