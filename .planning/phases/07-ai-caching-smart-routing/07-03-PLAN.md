  <name>Task 1: Refactor route handler with three-tier routing</name>
  <files>src/app/api/chat/route.ts</files>
  <action>
Refactor the existing POST handler to implement three-tier routing using the pre-seed cache approach. All requests flow through `streamText` -> `toUIMessageStreamResponse()` for consistent streaming protocol. Known topics are pre-seeded into the response cache so the caching middleware returns them as cache hits without calling Claude.

**Add these imports:**
```typescript
import { wrapLanguageModel } from "ai";
import { classifyIntent } from "@/lib/ai/intent-router";
import { getStaticResponse, textToStreamChunks } from "@/lib/ai/static-responses";
import { buildMinimalContext } from "@/lib/ai/topic-context";
import { cachingMiddleware } from "@/lib/ai/caching-middleware";
import { responseCache, hashMessage, DEFAULT_TTL } from "@/lib/ai/response-cache";
```

**Create the wrapped model at module level (created once):**
```typescript
const cachedModel = wrapLanguageModel({
  model: anthropic("claude-haiku-4-5-20251001"),
  middleware: cachingMiddleware,
});
```

**Refactor the POST handler:**

1. Rate limiting stays unchanged at the top.
2. `getEnv()` stays unchanged.
3. Parse messages from request body.
4. Extract last user message text from the messages array (filter for role "user", get last one, extract text parts).
5. Classify intent: `const intent = classifyIntent(lastText)`.
6. **Tier 1 pre-seed:** If intent is not "novel", hash the user message, check if already cached, and if not, pre-seed the response cache:
```typescript
if (intent !== "novel") {
  const cacheKey = hashMessage(lastText);
  if (!responseCache.get(cacheKey)) {
    const staticText = getStaticResponse(intent);
    const chunks = textToStreamChunks(staticText);
    responseCache.set(cacheKey, {
      chunks,
      rawCall: { rawPrompt: null, rawSettings: {} },
    }, DEFAULT_TTL);
  }
  // Fall through to streamText -- middleware will find cache hit
}
```
7. **Tiers 2 and 3** are handled by the caching middleware automatically when streamText runs.
8. Call `streamText` with `cachedModel` instead of the raw anthropic model:
```typescript
const result = streamText({
  model: cachedModel,
  system: buildMinimalContext(intent !== "novel" ? intent : null),
  messages: await convertToModelMessages(messages.slice(-20)),
  temperature: 0,
  maxOutputTokens: 1024,
});
return result.toUIMessageStreamResponse();
```
9. Keep existing error handling and rate limiting unchanged.

**Add debug logging** (gated behind development) to confirm which tier handles each request:
```typescript
const tierLabel = intent !== "novel" ? "static" : responseCache.get(hashMessage(lastText)) ? "cache-hit" : "api-call";
console.log(`[AI Route] intent=${intent}, tier=${tierLabel}`);
```
Place this AFTER the Tier 1 pre-seed block but BEFORE the streamText call.

IMPORTANT: Remove the old `buildSystemPrompt()` import and usage. Replace with `buildMinimalContext`. The old system-prompt.ts file stays in the codebase (other things may reference it) but route.ts no longer uses it directly.
