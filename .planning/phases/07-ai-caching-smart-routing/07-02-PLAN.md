Create an in-memory TTL cache following the same pattern as rate-limit.ts (Map-based, periodic cleanup).

**ResponseCache class:**
```typescript
interface CacheEntry<T> {
  value: T;
  expiresAt: number;
}

export class ResponseCache<T> {
  private store = new Map<string, CacheEntry<T>>();
  private maxSize: number;

  constructor(maxSize = 200) {
    this.maxSize = maxSize;
  }

  get(key: string): T | undefined { ... }  // Returns undefined if expired, deletes expired entry
  set(key: string, value: T, ttl: number): void { ... }  // Evicts oldest if at maxSize
  get size(): number { return this.store.size; }
}
```

For `set()`: if store size >= maxSize, delete the first (oldest) entry before inserting. This is a simple FIFO eviction -- good enough for a portfolio site.

Default TTL: 1 hour (3_600_000 ms). Export as `DEFAULT_TTL`.

**Hash utility:**
```typescript
import { createHash } from "crypto";

export function hashMessage(message: string): string {
  return createHash("sha256")
    .update(message.toLowerCase().trim())
    .digest("hex");
}
```

IMPORTANT: Hash ONLY the last user message text (normalized). Do NOT hash the full conversation history -- that would make every request unique (Pitfall 2 from research).

**Singleton instance:**
Export a pre-created instance: `export const responseCache = new ResponseCache<CachedStreamData>(200);`

Define `CachedStreamData` interface:
```typescript

  chunks: LanguageModelV3StreamPart[];
  rawCall: { rawPrompt: unknown; rawSettings: Record<string, unknown> };
}
```

Import `LanguageModelV3StreamPart` from `@ai-sdk/provider` (already available as transitive dependency, verified).


Create the LanguageModelV3Middleware that intercepts streamText calls to check/populate the response cache.

Import from the correct packages:
- `simulateReadableStream` from `"ai"`
- `LanguageModelV3Middleware`, `LanguageModelV3StreamPart` from `"@ai-sdk/provider"`
- `responseCache`, `hashMessage`, `DEFAULT_TTL`, `CachedStreamData` from `./response-cache`

The middleware implements `wrapStream` (primary -- used by streamText) and optionally `wrapGenerate` (for non-streaming calls, same pattern).

**wrapStream implementation:**
1. Extract the last user message text from params. Navigate params to find message content -- the params object contains `prompt` which is an array of messages. Find the last message with role "user" and extract its text content.
2. If no user message found, fall through to `doStream()` (no caching for edge cases).
3. Hash the user message text with `hashMessage()`.
4. Check `responseCache.get(cacheKey)`.
5. **Cache hit:** Return `{ stream: simulateReadableStream({ chunks: cached.chunks, chunkDelayInMs: 15 }), rawCall: cached.rawCall }`.
6. **Cache miss:** Call `const { stream, ...rest } = await doStream()`. Pipe stream through a TransformStream that collects chunks in `transform()` and calls `responseCache.set(cacheKey, { chunks, rawCall: rest.rawCall }, DEFAULT_TTL)` in `flush()`. Return `{ stream: stream.pipeThrough(transformStream), ...rest }`.

**Extracting user message from middleware params:**
The middleware params have a `prompt` field containing `LanguageModelV3Prompt` which is an array of `LanguageModelV3Message`. Each message has `role` and `content`. For user messages, content is an array of parts -- find parts with `type: "text"` and extract `text` field.

```typescript
function extractLastUserMessage(params: { prompt: LanguageModelV3Prompt }): string | null {
  const userMessages = params.prompt.filter(m => m.role === "user");
  const lastUser = userMessages[userMessages.length - 1];
  if (!lastUser) return null;
  const textParts = lastUser.content.filter(p => p.type === "text");
  return textParts.map(p => p.text).join(" ") || null;
}
```

Import `LanguageModelV3Prompt` from `@ai-sdk/provider` if available, otherwise type it inline.

Export the middleware: `export const cachingMiddleware: LanguageModelV3Middleware = { ... }`

IMPORTANT: Do NOT stringify the entire params object for hashing (Pitfall 4 from research). Only hash the user message text.
